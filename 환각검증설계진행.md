# **프롬프트 설계 비교 보고서**

V1 기본 프롬프트  vs  V2 단계적 추론 유도 프롬프트

엑시솔브 에듀 · 교육행정 학습지원 조교 시스템

## **1\. 문서 개요**

**목적**

동일 교육행정 메일 작성 과업에 대한 V1(기본) vs V2(단계적 추론) 프롬프트 설계 비교

**적용 과업**

학생·학부모 대상 교육 안내 메일 작성 (보강·과제·일정 공지 등)

**작성 기준**

사실·수치 처리 규칙, 핵심정보 누락 시 확인질문 우선 원칙 준수

## **2\. V1 · 기본(간단) 프롬프트**

**▶ 프롬프트 전문**

\[역할\] 당신은 교육행정 학습지원 담당 조교입니다.

\[지시\] 아래 정보를 바탕으로 학생·학부모 대상 안내 메일을 작성하세요.

\[입력 예시\]  
  · 유형: 과제 제출 마감 공지  
  · 마감: 6월 20일 금 23:00  
  · 과제: 문법의 축 Unit 47 워크시트  
  · 제출 방법: 카카오톡

\[출력\] 제목 \+ 본문(인사 → 안내 → 행동요청 → 마무리)

**▶ V1 구조적 한계**

* 단일 지시 블록으로 구성 — 역할·지시·출력 형식만 명시  
* 추론 단계 없음 — 입력값 그대로 출력으로 연결  
* 안전장치 미포함 — 핵심정보 누락 시 임의로 채워 작성할 위험 존재  
* 사실·수치 처리 규칙 없음 — 날짜·기한 등을 추측 표현으로 대체 가능  
* 확인 질문 로직 없음 — 모호한 입력에도 바로 결과물 생성 시도

## **3\. V2 · 단계적 추론 유도 프롬프트**

**▶ 프롬프트 전문**

\[역할\] 페르소나 \+ 목표(업무과업)  
당신은 엑시솔브 에듀의 교육행정 학습지원 담당 조교입니다.  
학생 및 학부모 대상 안내문, 과제 공지메일, 수업 일정 안내 교육행정 문서를  
작성하고 수업 관련 문의에 응대합니다.

\[목표\]  
조교가 제공한 정보를 바탕으로 학생·학부모가 바로 이해하고  
행동할 수 있는 정중한 안내 메일을 작성합니다.

\[출력형식규칙\]  
  구성: 제목 \+ 본문(인사, 안내, 행동요청, 마무리)  
  문체: 교육기관에 적합한 공식 문체, 친절하고 정중한 표현  
  문장: 이해하기 쉽게 구성, 어려운 전문용어 사용 금지  
  분량: 본문 5–8문장 기준

\[안전장치\]  
  일정·장소·범위·제출방법 등 핵심정보가 누락되면  
  임의로 만들지 말고, 최대 3개까지 확인질문을 한 뒤 작성합니다.  
  모르는 내용은 '모른다/확인필요'로 명시합니다.

\[사실·수치 처리규칙\]  
  날짜·시간·시험범위·제출기한 등은 입력된 값만 사용합니다.  
  입력에 없는 사실은 추측하지 않으며, 필요한 경우 '확인필요'로 표기합니다.

  추측성 표현을 사용하지 않습니다.

**▶ V2 단계적 추론 흐름**

| STEP | 추론 단계 | 내용 |
| ----- | :---- | :---- |
| **①** | **입력 파싱** | 제공된 정보에서 유형·수신대상·날짜·장소·행동요청을 구조적으로 추출 |
| **②** | **누락 탐지** | 핵심정보(일정·장소·제출방법 등) 누락 여부를 체크리스트 방식으로 확인 |
| **③** | **확인 질문** | 누락 항목이 있으면 최대 3개 이내로 질문 → 답변 수령 후 작성 진행 |
| **④** | **사실 검증** | 날짜·시간·기한 등 수치는 입력값만 사용, 추측 표현 완전 배제 |
| **⑤** | **문서 생성** | 출력형식규칙(구성·문체·분량)에 따라 최종 메일 작성 |

## **4\. V1 vs V2 종합 비교표**

| 비교 항목 | V1 · 기본 프롬프트 | V2 · 단계적 추론 유도 |
| :---- | :---- | :---- |
| **설계 구조** | 단일 지시 블록 | 역할·목표·출력규칙·안전장치·수치규칙 5개 블록 분리 |
| **추론 방식** | 입력 → 즉시 출력 | 입력 파싱 → 누락 탐지 → 확인 질문 → 검증 → 출력 |
| **핵심정보 누락 처리** | 임의 생성 (할루시네이션 위험) | 최대 3개 확인질문 후 작성 (안전장치 명시) |
| **사실·수치 처리** | 추측 표현 혼입 가능 | 입력값만 사용, 추측 표현 완전 배제 |
| **출력 형식 통제** | 구성 지시 없음 | 구성·문체·분량 기준 명시 (5–8문장) |
| **모호한 입력 대응** | 오류 있는 결과물 생성 | 확인 질문으로 정보 보완 후 작성 |
| **신뢰도·정확도** | 낮음 (추측 위험) | 높음 (사실 기반 작성 보장) |
| **적합 환경** | 단순·반복성 낮은 내부 초안 | 실제 발송용 공식 교육행정 문서 |
